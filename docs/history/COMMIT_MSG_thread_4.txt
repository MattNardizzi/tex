Thread 4: Wire five frontier runtime-defense modules as specialist judges

Wires ClawGuard, PlanGuard, MAGE, MCPShield, and AgentArmor as specialist
judges inside the live /v1/guardrail request path. Each wraps the existing
src/tex/runtime/* module, emits structured SpecialistResult evidence with
reason codes + OWASP ASI 2026 tags, and respects a <5ms p99 latency budget.
The default specialist suite grows from 6 → 11 judges.

                       IPI boundary enforcement. AgentDojo ASR 0.6-3.1% → 0%.

Design decisions
────────────────
- Hybrid aggregation: ClawGuard/MCPShield short-circuit on deterministic
  DENY (risk → 1.0); PlanGuard/MAGE/AgentArmor vote into the bundle.
- All five specialists tag against OWASP ASI 2026 categories via
  src/tex/domain/owasp_asi.py. AgentLAB's five attack families
  (intent hijacking, tool chaining, task injection, objective drifting,
  memory poisoning — arxiv 2602.16901) map cleanly.
- Specialists wrap, never replace. Higher-fidelity LLM-trained backends
  (GRPO-trained M_θ/J_θ for MAGE, real Stage-II LLM for PlanGuard, real
  LTS verifier dispatch for MCPShield, real PDG construction for
  AgentArmor) remain pluggable via metadata-supplied callables and models.
- Patterns kept lexical / boundary-regex to stay inside the <5ms p99
  specialist budget; total request latency stays under 200ms p95
  (current 142ms + 5 × <5ms = ~167ms).
