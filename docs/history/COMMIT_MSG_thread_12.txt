Thread 12: composite TDX + NVIDIA GPU TEE attestation, bound per-decision

WHAT
====
Adds hardware-rooted composite TEE attestation to every /v1/guardrail
decision evidence record. When TEX_TEE_MODE=1, Tex collects Intel TDX
evidence AND NVIDIA H100/H200/B200 GPU attestation evidence, submits
both to Intel Trust Authority's /appraisal/v2/attest endpoint via
ITAConnector.get_token_v2(tdx_args, gpu_args), and embeds the resulting
PS384-signed JWT inside the canonical decision evidence record —
cryptographically bound through the existing SHA-256 payload chain.

A new POST /v1/tee/verify endpoint lets relying parties (insurers,
regulators, downstream agents) verify the JWT independently fail-closed.
The verification result carries an AR4SI trustworthiness vector per
draft-ietf-rats-ear-03 (Mar 15 2026).

NEW MODULES
===========
* src/tex/tee/composite.py — domain models (CompositeAttestationEnvelope,
  EatAiClaims, CompoundAttestationLink, TrustworthinessVector,
  CompositeVerificationResult, CpuTeeType, GpuTeeType, EatAiDigest).
  All pydantic v2 frozen+extra=forbid.

* src/tex/tee/tdx_attestation.py — Intel TDX evidence collector. Production
  path imports inteltrustauthorityclient.tdx.intel.intel_tdx_adapter and
  calls IntelTDXAdapter.collect_evidence(nonce=...) on a TDX-capable host
  (/sys/kernel/config/tsm/report or /dev/tdx_guest present). Dev-mode
  fallback returns a deterministic stub flagged is_dev_mode=True.

* src/tex/tee/h100_attestation.py — NVIDIA Attestation SDK binding.
  Production path uses nv_attestation_sdk.attestation.Attestation with
  local-GPU verifier. _detect_hwmodel_or_fallback() reads NVML to
  distinguish GH100 / GH200 / GB200 / GB300.

* src/tex/tee/attestation_client.py — composer (compose_attestation,
  compose_from_evidence), verifier (verify_attestation), CrossGuard
  decision_bound_nonce, test-mode JWT builder. Algorithm-agile signature
  verification routes through tex.pqcrypto.algorithm_agility for
  ML-DSA-65/87 and hybrid-ml-dsa-65-ed25519; falls back to the local
  cryptography library for PS384/RS256/ES384/ES256 (the algorithms ITA
  currently uses).

* src/tex/tee/__init__.py — clean public surface.

* src/tex/api/tee_routes.py — POST /v1/tee/verify and GET /v1/tee/status
  FastAPI router. The router uses the same verifier as Tex's internal
  evidence pipeline so external auditors hit the same code path.

===========================

    - Production mode + dev-stub evidence -> RuntimeError.
    - Verifier emits stable reason codes on every failure
      (nonce_mismatch, issuer_mismatch, expired, tdx_debuggable,

THREAD 12+ — MAY 2026 BLEEDING-EDGE SOTA AUGMENTATIONS
================================================================================

Adds src/tex/tee/sota_2026.py — augmentation block layered on top of the
canonical composite envelope. Every sub-augmentation implements a
published-but-not-yet-shipped May 2026 frontier:

1.  draft-ietf-rats-eat-measured-component-12 (IESG Last Call passed Jan 26
    2026, becoming Proposed Standard) — MeasuredComponent for every agent
    artifact loaded (model weights, policy bundles, retrieval indices,
    fine-tuned adapters, tokenizers). Tex profile bits for component-type,
    in_tee_memory, trusted_by_signers, immutable_during_session.

2.  draft-ietf-rats-corim-10 (March 2 2026) — CoRimReferenceValue triples
    (subject_class_id, predicate, object_digest) so a downstream Verifier
    can do offline policy match without fetching a separate CoRIM CBOR.

3.  draft-ietf-cose-dilithium-11 (Nov 15 2025) — IANA-requested COSE alg
    numbers: -48 ML-DSA-44, -49 ML-DSA-65, -50 ML-DSA-87.

4.  draft-ietf-jose-pq-composite-sigs-01 (Feb 27 2026) — verbatim hex byte
    labels: "COMPSIG-MLDSA65-Ed25519-SHA512" and
    "COMPSIG-MLDSA87-Ed448-SHAKE256".

5.  GpuTeePlatform extension — VERA_RUBIN_NVL72 (May 2026, world's first
    rack-scale CC), BLACKWELL_B300, BLACKWELL_GB200/GB300, JETSON_AGX_THOR
    (CC + TDISP), RTX_PRO_6000 (R580 TRD1 GA).

6.  DriverPinning — defence against attestation-replay-after-driver-downgrade.
    NVIDIA R590 TRD1 (Dec 2025) is the first GA build supporting Blackwell
    PPCIE multi-GPU with the IV-exhaustion fix.

7.  TdispEvidence — PCIe SIG TDISP (TEE Device Interface Security Protocol).
    Closes the gap between CPU-TEE memory protections and GPU DMA. Required
    lock_state = run-locked for production. Jetson AGX Thor ships TDISP; B200/
    B300 server SKUs gaining support throughout 2026.

8.  MultiGpuBatch — Intel Trust Authority composite v2 supports up to 8 GPUs
    per request. Batch summary lets policy match against the full set.

9.  PersistentMemoryRegion — addresses arxiv 2605.03213 §VI (Forough et al.,
    May 7 2026): "end-to-end protection of persistent agent memory remains
    an open problem". Tex carries SHA-3-256 digests of vector stores, KV
    caches, fine-tuned adapters, tool state, session transcripts, and
    long-term memory regions inside the same attestation envelope.

10. ScittReceipt — draft-ietf-scitt-architecture-22 (Oct 10 2025). Receipt
    from a SCITT Transparency Service binding the composite envelope to a
    position in an append-only Merkle tree. Microsoft Code Transparency
    Service (CCF / COSE_Sign1) shape supported.

11. TcbAdvisoryCheck — Intel publishes a continuous stream of TDX security
    advisories. Operator blocklist via TEX_TEE_BLOCKED_ADVISORY_IDS env var
    fails-close on a named advisory faster than Intel's own TCB-R cadence.

12. TsmEventLog — Linux 6.7+ TSM ConfigFS event log binding. Full
    cryptographic chain "kernel→RTMR→ITA" — the verifier can replay the
    event log into RTMR0..3 and check the ITA-signed RTMR values.

13. LongHaulNonce — three-nonce CrossGuard binding (decision + transcript +
    fleet) defeating transcript-tampering attacks and cross-deployment
    replay (CrossGuard §6.2). Composite nonce goes into the ITA report-data.
