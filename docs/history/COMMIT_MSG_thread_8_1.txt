thread 8.1: frontier upgrades — BLAKE3-ML-DSA, AIR eradication, NP monitor selection

Three bleeding-edge primitives that exist in published work as of
May 19, 2026 but are not shipped in any AI-governance product:

1. BLAKE3-accelerated ML-DSA-B (Project Eleven + Taurus, Oct 2025)
2. AIR-style LLM-synthesized eradication rules (arxiv 2602.11749, Feb 2026)
3. Neyman-Pearson optimal multi-monitor selection (Hua et al. NeurIPS 2025)

Wired end-to-end. 69 new tests; 2,637/2,637 regression passing.

--- Frontier #1: BLAKE3-ML-DSA-B ---

src/tex/pqcrypto/blake3_ml_dsa.py — Blake3MlDsaProvider implements
FIPS 204 §5.4 HashML-DSA with BLAKE3 as the pre-hash function. Domain
tag b"tex-ml-dsa-b/v1\0" (16 bytes) ensures clean algorithm binding.
Pre-hash: BLAKE3(domain_tag || length_le8 || message) → 32-byte
digest → fed to stock ML-DSA via existing MlDsaProvider. Captures the
dominant pre-hashing performance win (15-30% per Project Eleven).
Real, deployable, FIPS-204-compliant.

src/tex/pqcrypto/algorithm_agility.py — added BLAKE3_ML_DSA_65 to the
SignatureAlgorithm enum. Dispatcher routes to Blake3MlDsaProvider.

src/tex/institutional/_pq_signing.py — promoted BLAKE3_ML_DSA_65 to
top of the selection chain (now: BLAKE3-ML-DSA → ML-DSA-65 → HYBRID →
ECDSA-P256). Bumped selection_chain_version to "v2-blake3-thread-8.1".
Probe updated to recognise BLAKE3 family as needing liboqs probe.

Honest engineering note: pure Project Eleven Rust replaces SHAKE
*inside* the lattice algorithm. We implement the standard-compliant
FIPS 204 §5.4 HashML-DSA subset (BLAKE3 pre-hashing only, delegating
to liboqs for the lattice math). This captures the dominant pre-hash
performance advantage per the Taurus blog ("the pre-hashing advantage
for ML-DSA-B remains significant, especially for larger message
sizes"). The full Project Eleven design is flagged as a future option
pending Python binding availability.

Tests: tests/pqcrypto/test_blake3_ml_dsa.py — 16 tests, MagicMock
verifies pre-hashing without needing liboqs.

--- Frontier #2: AIR eradication rule synthesis ---

src/tex/intervention/kinds.py — added InterventionKind.ERADICATION_
RULE_SYNTHESIS as the 8th kind.

src/tex/intervention/eradication.py — EradicationRuleSynthesizer with
two modes (LLM-preferred + deterministic fallback), IncidentContext
and SynthesizedRule frozen dataclasses, LLMClient Protocol,
InMemoryRuleRegistry with idempotent register + pattern matching,
RuleRegistry Protocol. Plan-level checks pipeline: schema, safety
(forbid-only structurally prevents widening), cost (predicate_count
≤ 10, ltlf_depth ≤ 6, both operator-tunable). LLM mode failure or
plan-check rejection of LLM rule → deterministic fallback.

src/tex/intervention/engine.py — added _PHASE_ERADICATE AIR phase
mapped to ERADICATION_RULE_SYNTHESIS. Constructor accepts new optional
params eradication_synthesizer + rule_registry. Apply path detects the
kind, synthesises + registers, embeds serialisable rule dict in
governance-log payload as "synthesised_rule". FAIL-CLOSED when synth/
registry not wired or incident_context malformed. ERADICATION_RULE_
SYNTHESIS is NOT in blocking_kinds → defaults to SANCTION verdict
(admit current event, rule blocks future recurrences) matching AIR §3.

src/tex/intervention/__init__.py — exports added.

Tests: tests/intervention/test_eradication.py — 30 tests covering
deterministic-mode severity escalation, LLM-mode JSON parsing +
fallback on every failure mode, plan-check rejection of overcomplex
LLM rules with deterministic fallback firing, registry idempotence +
pattern matching + wildcard actor, full engine-integration round trip
with audit log embedding, FAIL-CLOSED on bad incident_context.

--- Frontier #3: Neyman-Pearson monitor portfolio selection ---

src/tex/intervention/neyman_pearson.py — MonitorPortfolio (frozen,
with from_rates builder), NeymanPearsonSelector (greedy Lagrangian
knapsack with operator-chosen lambda, deterministic tie-break by
monitor_id), PortfolioSelection result dataclass, MonitorCandidateSource
Protocol, compose_intervention_pool() (union of selected monitors'
candidates with dedup). Algorithm: utility = log(LR_m) - λ·cost_m,
greedy descending, accept if utility > 0 AND fits budget AND union-
bound FA ≤ alpha.

Smoke test verified: 3 monitors (drift LR=28.3 cost=2, contracts
LR=95 cost=1, weak LR=15 cost=5), budget=4, α=0.05 → selected
{contracts, drift}, composite_detection=0.9925, composite_FA=0.0397.

src/tex/intervention/__init__.py — exports added.

Tests: tests/intervention/test_neyman_pearson.py — 23 tests covering
from_rates LR computation, validation rejections, frozen shape,
alpha enforcement via union bound (two FA=0.04 monitors → only one
selected since 1-(0.96)² = 0.0784 > 0.05), Lagrangian cost-aversion
responds to lambda, deterministic tie-break, composite detection
independence-union formula, broken-source skip.

--- Algebraic typo fix from Thread 8 still applies ---

The AAF paper's printed Proposition 1 formula λ_min = (g_max+αH)/(Hη*)
does not algebraically rearrange Theorem 5. Tex implements the
algebraically-correct form λ_min = g_max/H + α/η*. Documented in the
bounded_compromise module and verified by the round-trip property
test in tests/intervention/test_bounded_compromise.py. Thread 8.1
does not touch this code path.
