Thread 13: VET Web Proofs + Agent Identity Document with PQ-default selective disclosure

WHAT
====
Closes the audit-runtime gap for third-party AI API calls and ships
the first agent-governance Agent Identity Document with post-quantum
default + bbs-2023-shape selective disclosure.

Tex now notarizes every closed-model API call (OpenAI, Anthropic,
Google, Mistral) with attestor-signed Web Proofs over the TLS session.
The proofs are independent of the API provider's cooperation — an
auditor verifies them against the recorded server cert fingerprint and
the attestor's public key, not against any provider-side log.

Every Tex-managed agent now carries a W3C Verifiable Credentials 2.0
Agent Identity Document. The AID's base proof is signed under ML-DSA-65
(FIPS 204, NIST L3) by default via tex.pqcrypto.algorithm_agility —
Tex's post-quantum hedge against the classical Ed25519 keys that ship
in every competing agent-identity stack as of May 2026 (Microsoft Agent
Governance Toolkit Agent Mesh, Indicio ProvenAI, walt.id Enterprise
Stack, Microsoft Entra Verified ID). Selective disclosure is
implemented over salted-commitment Merkle trees in the shape of the
W3C bbs-2023 Candidate Recommendation, with per-presentation
holder-binding HMAC for replay resistance.

The k-of-n MultiAttestorCommittee is Tex's wedge: no AI-governance
vendor (Microsoft AGT, Zenity, Noma, Pillar, Lakera, Protect AI, Rubrik
SAGE) ships threshold third-party API notarization. Reclaim Protocol
runs a decentralized attestor network but exposes only single-signature
results; Tex demands k-of-n and surfaces the threshold in the audit
record.

NEW MODULES
===========
* src/tex/vet/selective_disclosure.py — Pure-Python implementation of
  the bbs-2023-shape selective-disclosure primitive. RFC 6962-style
  binary Merkle tree with correct odd-leaf duplication (debugged at
  arbitrary sizes 1, 2, 3, 4, 5, 7, 8, 12, 17, 33). Salted disclosure
  tuples per the SD-JWT VC construction
  (draft-ietf-oauth-sd-jwt-vc-16 §4.2.1). Issuer signature routed
  through tex.pqcrypto.algorithm_agility; defaults to ML-DSA-65. Per-
  presentation HMAC binding for audience+nonce replay protection. The
  cryptosuite name emitted in the base proof is
  "bbs-2023-shape-{algorithm}" — drop-in compatible with a future
  swap-in of a real BBS+ pairing implementation when py-ecc adds
  robust F_p^2 sqrt for G2 decompression.

* src/tex/vet/web_proofs.py — WebProof, WebProofAttestation,
  WebProofMode enum (ZKTLS_RECLAIM, ZKTLS_PLUTO, TLSNOTARY_MPC,
  MULTI_ATTESTOR, STUB), ZkTlsAttestorClient (Reclaim attestor-core
  HTTP API + Pluto Labs notary), TlsNotarySubprocessClient (subprocess
  to the Rust binary at $TEX_TLSNOTARY_BIN), MultiAttestorCommittee,
  notarize_session(), verify_web_proof(). Backwards-compatible with the
  VET-paper notarize_session(target_host, session_log) signature;
  extended with mode selection, committee, cert pinning, headers, and
  trusted-attestor-pubkey allowlists. When no live attestor is
  configured, falls to a clearly-marked STUB mode flagged in the proof.

* src/tex/vet/agent_identity_document.py — Full Pydantic-v2
  AgentIdentityDocument model (replaces the dataclass stub). Issuance
  via issue(request: AidIssuanceRequest), verification via
  verify(aid: AgentIdentityDocument), selective-disclosure
  presentation via present(aid, request: AidPresentationRequest), and
  W3C VC 2.0 envelope rendering via to_vc_2_0(aid). Mandatory
  disclosures (agent_id, issuer_did, aid_spec) always revealed;
  everything else selectively disclosable. Presentation header binds
  audience+nonce+iat+exp; HMAC-bound to the Merkle root so replays
  against a different audience fail closed.

* src/tex/vet/ptv_attestation.py — First known Python implementation of
  draft-anandakrishnan-rats-ptv-agent-identity-00 (April 5, 2026) and
  draft-anandakrishnan-ptv-attested-agent-identity-00 (March 31,
  2026). Encodes the JSON envelope shape with the
  groth16-2026/plonk-2026 PtvAttestationMethod registry. Ships a
  Schnorr-bridge fallback that uses real ML-DSA-65 or Ed25519
  signatures over the canonical envelope — the wire format is
  identical so a future real Groth16-2026 prover (e.g. ark-circom,
  ezkl, Tokamak SP1 zkVM) is a drop-in replacement behind
  generate_ptv_attestation().

* src/tex/vet/aivs_micro.py — draft-stone-aivs-00 (March 2026,
  SwarmSync.AI) AIVS-Micro continuous-monitoring stub. Ed25519-signed
  per the spec's mandate. emit_aivs_micro() + verify_aivs_micro()
  with tamper detection and ±5min clock-skew tolerance. The W3C AIVS
  Community Group launched April 5, 2026; this is the first known
  Python implementation.

* src/tex/vet/sd_jwt_vc.py — draft-ietf-oauth-sd-jwt-vc-16 (April 24,
  2026, latest) + draft-nandakumar-agent-sd-jwt-02 (Feb 28, 2026)
  SD-Card format. Tracks the latest revision (existing libraries
  track -13). Per-claim SdJwtClaimVisibility (ALWAYS, ALLOWED, NEVER)
  matches the new Claim Selective Disclosure Metadata section.
  Key-binding JWT (KB-JWT) support for holder authentication.

* src/tex/vet/txn_tokens.py — draft-oauth-transaction-tokens-for-agents-06
  (April 11, 2026, Raut, Amazon). TxnTokenScope (RAR-style binding),
  TxnTokenClaims (act/sub per the latest revision), TxnTokenArtifact
  (compact JWS), 60-second TTL default, ML-DSA-65 signing default.
  Implements the April 30, 2026 Five Eyes joint guidance's
  "short-lived OAuth 2.0/OIDC tokens" requirement for agent
  authentication.

* src/tex/vet/registry.py — AidRegistry Protocol, InMemoryAidRegistry
  thread-safe in-memory implementation with optional AidRegistryMirror
  hook for Postgres durability (best-effort writes that NEVER block
  the in-memory operation, mirroring the EvidenceRecorder design from
  Thread 5).

* src/tex/vet/integration.py — The integration hook into the
  /v1/guardrail evidence path. attach_web_proof_to_payload() wraps a
  WebProof into a decision payload; verify_payload_web_proof()
  verifies it round-trip. Fail-open on attach (an unreachable attestor
  must not block decisions); fail-closed on verify (an invalid proof
  is treated as tampering).

* src/tex/api/vet_routes.py — FastAPI router with 10 endpoints:
  POST /v1/vet/issue-aid, POST /v1/vet/verify-aid,
  POST /v1/vet/present-aid, POST /v1/vet/verify-presentation,
  GET  /v1/vet/aid/{agent_id}, POST /v1/vet/update-aid-status,
  POST /v1/vet/notarize, POST /v1/vet/verify-web-proof,
  POST /v1/vet/issue-txn-token, POST /v1/vet/verify-txn-token. All
  Pydantic-v2 frozen+extra=forbid request/response models.
