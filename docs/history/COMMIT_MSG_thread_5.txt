Thread 5 — C2PA Content Credentials → Evidence Emission (wired)

Wires the previously built-but-unwired ``tex.c2pa`` package into the
``EvidenceRecorder.record_decision`` path so every PERMIT verdict on
an outbound AI-generated artifact emits a C2PA 2.4 Content Credential
with a Tex post-quantum evidence cosign. FORBID verdicts with a
refusal reason emit a SCITT-shaped refusal event per
``draft-kamimura-scitt-refusal-events-02``.

What this thread implements
---------------------------

* ``tex.c2pa.evidence_emission`` — ``build_signed_manifest_with_cosign``
  produces the outer COSE_Sign1 (ES256 / Ed25519, C2PA 2.4 §13.2
  allow-list compliant) plus a ``tex.evidence_cosign`` extension
  assertion signed under ``tex.pqcrypto.algorithm_agility`` with
  ML-DSA-65 by default (NIST FIPS 204, NIST Security Level 3).
  Ed25519 fallback for environments without liboqs.
* ``tex.c2pa.cosign_verifier`` — verifies the cosign and reports
  per-attack defense status for the five attack classes from
  arxiv 2604.24890.
* ``tex.evidence.c2pa_emitter`` — thin facade so the recorder does
  not pay the c2pa import cost when not in use.
* ``tex.evidence.manifest_mirror`` — ``PostgresManifestMirror``
  backing the new ``tex_evidence_manifests`` table.
* ``tex.evidence.recorder`` — ``record_decision`` takes optional
  ``outbound_artifact`` and ``c2pa_context``; pre-Thread-5 callers
  see zero change.
* ``tex.api.c2pa_routes`` — ``GET /v1/evidence/{record_id}/c2pa``
  and ``POST /v1/c2pa/verify``.

Six-attack-defense semantics
----------------------------

The ``tex.evidence_cosign`` assertion closes the six attack classes
in arxiv 2604.24890 §"Key Findings":

  1. timestamp_swap                — bound_timestamp in signed cosign input
  2. revocation_skipped            — revocation_proof field (CRL hash pin)
  3. cross_validator_contradiction — canonicalization_version field
  4. exclusion_range_tamper        — full_file_sha256 over entire asset
  5. cert_expiry_before_retention  — retention_anchor into hash chain
  6. conformance self-reporting    — addressed operationally

Caveats
-------

  * COSE codepoint for ML-DSA is still draft, so the outer
    COSE_Sign1 is ES256/Ed25519 (spec-conformant). PQ coverage is
    supplied by the cosign assertion under the outer signature.
  * Vanilla C2PA validators (Adobe Inspect, CAI Verify,
    c2patool, Microsoft Edge) treat the cosign as an unknown
    extension assertion — spec-defined behaviour. Tex-aware
    validators (or the ``POST /v1/c2pa/verify`` endpoint) report
    the six-attack-defense status explicitly.
  * The pre-existing ``tex.events.crypto_provenance`` ↔
    ``tex.ecosystem.engine`` circular import surfaces during test
    collection. ``tests/conftest.py`` now preloads
    ``tex.ecosystem`` to resolve the order. Out of Thread 5 scope
    to refactor.
