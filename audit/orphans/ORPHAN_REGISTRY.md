## Methodology

For every file in `src/tex/`, we:

1. AST-parse every Python file in `src/tex/`, `tests/`, `scripts/`, `sdks/`
2. Build a directed import graph including transitive `__init__.py` triggers (importing `tex.X.Y` triggers `tex.X.__init__.py` loading)
3. BFS-reach from `src/tex/main.py` → the **WIRED** set (377 files)
4. BFS-reach from every test file → the **TEST_REACHABLE** set
5. BFS-reach from every script/sdk file → the **SCRIPT_REACHABLE** set
6. Classify each file by which sets it appears in

## Section 5: What this means

Every file labeled TEST_ONLY has a test that exercises it directly. The tests pass. The code is correct in isolation.

What's missing is **integration call sites** — places where `main.py` or `commands/evaluate_action.py` or `engine/pdp.py` actually invokes these modules.

The highest-impact integration call sites to add (in priority order):

1. **`vet/integration.py:attach_web_proof_to_payload`** invoked from `commands/evaluate_action.py` after semantic-layer call
2. **Compliance emitter registry** invoked from `commands/evaluate_action.py` after `recorder.record_decision`
3. **`governance/kernel_mcp/syscall_gate.py`** wrapped around every `api/mcp_server.py` JSON-RPC `tools/call`
4. **`enforcement/` package** — decide whether this is the SDK path or runtime path

Each of these would convert ~200-1,800 lines of TEST_ONLY code to WIRED.
