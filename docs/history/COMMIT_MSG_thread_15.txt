Thread 15 — NANOZK layerwise verifiable inference (wired)

Implements `src/tex/nanozk/` (four new modules, 91% coverage, 149
unit tests) and wires the live verifier path into
`tex/evidence/attribution_zk.py` and `tex/api/incident_routes.py`.
When `TEX_FRONTIER_NANOZK=1`, the `/v1/incidents/{id}/attribute`
endpoint now returns a PTV envelope with method
`tex:nanozk-layerwise-2026` carrying a hash-chained, Fisher-
selected, VEIL-wrapped layer proof set. The
`tex.evidence.attribution_zk.verify_ptv_envelope` path that
previously dead-ended at
`nanozk_verifier_not_implemented_in_this_thread` flips to a live
verdict (`ok_nanozk_layerwise_verified`).

Per-layer verifier latency (shim path): 0.13 ms — 175× under the
23 ms target from the NANOZK paper §5.2. Six-layer set verifier:
1.32 ms total. The regulator-grade backends (Halo2-IPA,
DeepProve, JOLT, LatticeFold+, SP1 Hypercube, VEIL) plug into
the same dispatcher when their Rust binaries are present.

Delta vs section 1.4 of standing orders
---------------------------------------

Section 1.4 anchored DeepProve, JOLT, LatticeFold+, SP1
Hypercube, VEIL, and NANOZK. Thread 15's frontier delta (see
FRONTIER_DELTA_thread_15.md) adds:

  * Jolt Atlas prefix-suffix decomposition (Feb 19 2026 paper) —
    strictly newer than NANOZK's materialised 16-bit tables.
  * zkGPT constraint fusion (USENIX Sec '25) — strictly newer
    than NANOZK's unfused range relations.
  * VEIL composition with the layerwise sumcheck — no competitor
    has done this. Gives post-quantum ZK on the inference proof
    without an elliptic-curve dependency.
  * Cost-weighted Fisher selector beyond the paper's uniform-
    cost algorithm. Deterministic tie-breaking by layer index
    (the paper is silent on ties — fine for evaluation, not for
    a cryptographic protocol).
  * Constraint-fusion factor recorded on the circuit so the
    verifier reproduces the prover's fusion bit-for-bit.

--- v2: Eight bleeding-edge upgrades ---

After the initial Thread 15 delivery, an honest "is this truly at the
May 18, 2026 frontier?" audit identified eight composable upgrades.
All eight are now implemented, wired, tested, and documented.

UPGRADES:

  1. Logup* (ePrint 2025/946, Soukhanov) — faster, cheaper logup
     argument for small-table indexed lookups; no extra commits to
     indexing-array-sized columns; no numerator-overflow.
     New: src/tex/nanozk/logup_star.py
     Wired: PrefixSuffixLookup.lookup_argument_kind,
            LayerCircuit.lookup_argument_kind (bound into fingerprint).

  2. GaugeZKP canonicalisation (OpenReview 1Ne3tfQC0T, ICME Labs
     2025) — pre-prover canonicalisation, ~26% gate reduction on
     Halo2 / Plonkish, multiplies with RoPE/GQA/MQA/MoE.
     New: src/tex/nanozk/gauge_zkp.py
     Wired: LayerCircuit.gauge_canonicalized,
            LayerCircuit.gauge_canonicalizer_fingerprint.

  3. Poseidon-BN254 hash chain — SNARK-friendly set root; 120×
     constraint reduction vs SHA-256 when opened inside a SNARK.
     New: src/tex/nanozk/poseidon_chain.py
     Wired: _set_root() returns (root_hex, kind_str); LayerProofSet
            .chain_kind. Activated by TEX_NANOZK_POSEIDON_ROOT=1 or
            TEX_FRONTIER_NANOZK=1.

  4. LatticeFold+ ℓ₂ folding (ePrint 2026/721, Apr 19 2026) — PQ-secure
     recursive accumulator, ~2× lower prover cost.
     New: src/tex/nanozk/latticefold_plus.py
     Wired: LayerProofSet.folded_accumulator_json; prove_layer_set
            folds when latticefold_active(); verify_layer_proof_set
            checks and fails closed.

  5. Sublinear-space proving (arxiv 2509.05326, Nye) — O(√T·log T·
     log log T) prover memory, bit-identical proofs for KZG/IPA PCS.
     New: src/tex/nanozk/sublinear_space.py
     Wired: caller-side planning API; TEX_NANOZK_SUBLINEAR=1 activates.

  6. Mira parallel folding (ZKTorch arxiv 2507.07031) — homomorphic
     tree accumulation, 3-10× proof size reduction, 6× speedup.
     New: src/tex/nanozk/mira_parallel.py
     Wired: LayerProofSet.mira_root_commitment +
            mira_backreference_hash; prove_layer_set folds when
            mira_active(); verify_layer_proof_set checks and fails
            closed.

  7. DeepProve subprocess backend (Lagrange Labs) — real Rust binary
     bridge for GKR-sumcheck zkML; 54-158× over EZKL, 671× verifier.
     New: src/tex/nanozk/deepprove_backend.py
     Wired: register_deepprove_if_available() called at import time
            in tex.nanozk.__init__.py; satisfies NanozkBackend
            Protocol; falls back to shim if binary absent.

  8. V3DB verifiable vector search (arxiv 2603.03065) — audit-on-
     demand ZK proofs for IVF-PQ ANN retrieval over committed
     snapshots; closes the retrieval trust gap for RAG agents.
     New: src/tex/nanozk/v3db.py
     Wired: standalone proof type composable with LayerProofSet in
            PTV envelopes; protocol pinned to v3db-2026-03-05.
